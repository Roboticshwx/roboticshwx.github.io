📬ESP32串口通信

基础知识

串口是单片机常用的一种通信方式，通过两条线：TX和RX进行通信。

连接方式：一端的Tx接另外一端Rx，一端的Rx接另外一端Tx。

📮ESP32SPI通信

SPI 串行外设接口 (Serial Peripheral Interface) 是一种高速的，全双工，同步的通信总线。

ESP32集成了4个SPI控制器，分别是SPI0, SPI1, SPI2, SPI3，由于flash使用SPI0, SPI1，因此SPI2和SPI3是通用的SPI控制器，日常使用这两个SPI进行开发。

SPI2也叫HSPI，SPI3也叫VSPI。

使用默认的引脚，可以使用80MHz的时钟。

SPI2 (HSPI) SPI3 (VSPI) SPI Signal Description
13 23 MOSI 主设备输出，从设备输入
12 19 MISO 主设备输入，从设备输出
14 18 SCLK 时钟信号
15 5 SS 片选信号（用于从设备选择）

🦾ESP32输出PWM信号

ESP32的芯片中有专门用于脉宽调制（PWM）的模块, pwm信号在ESP32中有两个模块经常用到：

  • LEDc模块
    • 16个可用通道;
    • 8个可用定时器;
    • 频率范围从1Hz~40MHz;
    • PWM分别率从1位到16位;
  • MCPWM (Motor Control PWM)
    • 6个可用通道;
    • 内置死区控制, 用于电机控制的驱动H桥;
    • 包括捕获功能, 可以读取外部PWM信号的频率和占空比;
    • 主要用于复杂的电机控制;

⚙️ESP32常用模块

超声波模块

以HC-SR04为例：

原理：

  1. 单片机给模块发送一个至少10us的高电平脉冲到Trig引脚（触发）。
  2. 模块接收到脉冲后会在Echo引脚（回声）发出一个高电平脉冲。
  3. 这个脉冲的持续时间就是声波从发出到反射回来的时间，通过测量声速往返的时间，便可以得到距离。

蓝牙模块使用的是<BLEDevice.h>这个库，通过一段示例代码，使得手机或者平板可以通过蓝牙设备搜索，以LED为信号，来尝试连接esp32;

UUID用于区分一个服务器中不同的服务和不同的特性，可以用UUID生成器（直接搜一个）来生成UUID;

在BLE中，有以下几个重要的概念：

  • BLE设备 (BLE Device):

    • 任何能够进行BLE通信的物理设备都可以被认为是一个BLE设备。这包括手机或平板或ESP32。

  • BLE服务器 (BLE Server):

    • 在BLE通信中，服务器是保存并提供数据的设备。服务器拥有一种称为“GATT (Generic Attribute Profile)”的数据结构，这个数据结构包含了这个服务器可以提供的所有数据。

  • BLE服务 (BLE Service):

    • 在GATT数据结构中，服务是一种组织数据的方式。每个服务都有一个唯一的UUID，以标识这个服务。服务包含了一组特性，每个特性都代表了一个数据点。

  • BLE特性 (BLE Characteristic):

    • 特性代表了一个数据点，例如一个传感器的读数或一个开关的状态。每个特性都有一个唯一的UUID，以标识这个特性。特性可以有不同的属性，例如可以读取、可以写入、可以通知等。

因此需要在代码中一步步创建以上四个对象。最后开始服务和开始广播，则可以在手机或者平板中找到对应的esp32设备进行连接。

WiFi模块

Esp32的WiFi模块有三种工作模式：

  • Wi-Fi Station
    • 作为网络的客户端，可以连接路由器与互联网交换数据；
  • Access Point
    • 作为一个“热点”，可以不需要路由器，其他设备连接Esp32与其交换数据；
  • Wi-Fi Station + Access Point
    • 两者功能兼具；

参考资料：ESP32 Useful Wi-Fi Library Functions (Arduino IDE)

🪄常见问题

无法定位到库

在项目工程文件夹中platformio.ini中添加lib_deps = ESP32Servo，然后重新编译。

多个库的话，用空格隔开

无法烧录

  • 检查串口是否被占用，或者是否连接到错误的串口；
  • 检查ESP32的pin2是否外部输入高电平，如果是则无法烧录；

中断函数

中断函数很难用，一般情况下不要使用，而且不要在中断函数中用串口发送信息。
