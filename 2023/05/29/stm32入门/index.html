开发环境搭建(点亮LED)

采用cortex-debug进行调试。

下载GCC ARM工具链

GNU Arm Embedded Toolchain Downloads

选择最新的版本，后缀为win32.exe的文件。

安装完成后将其添加到系统变量，勾选第三项;

在命令行中输入arm-none-eabi-gcc --version来验证是否安装成功并且添加到路径；

下载OpenOCD

Open On-Chip Debugger

解压完成之后需要手动将根目录下的bin文件夹添加到系统变量。

最后在命令行中输入 openocd --version 来检查 OpenOCD 是否安装成功并且添加到路径；

添加cortex-debug插件

在vscode中搜索这个插件，安装；

下载GNU make

MinGW Download

MinGW（Minimalist GNU for Windows）是一个可以在Windows环境下运行的GNU编译器套装，包括GCC（GNU Compiler Collection，GNU编译器套件）和其他一些开源工具。

通过下载MinGW来获取GNU make，下载对应的windows版本即可，记得添加路径。

在命令行中输入 make -v 来检查是否安装成功。

CubeMX用于生成stm32调试的基本代码，通过图形化的方式对stm32进行一个基础的硬件设置。

输出PWM

工作原理

STM32F1共有8个定时器，其中可以用于PWM输出的定时器有：

  • 高级定时器：TIM1, TIM8 (7路PWM)
  • 通用定时器: TIM2, TIM3, TIM4, TIM5 (4路PWM)

输出原理:

  • TIMx_ARR寄存器确定PWM频率；
  • TIMx_CCRx寄存器确定PWM占空比；
  • TIMx_CNT为定时器计数器；
  • PWM模式1：在向上计数时，TIMx_CNT < TIMx_CCR1时通道1为有效电平，否则为无效电平；在向下计数时，TIMx_CNT>TIMx_CCR1时通道1为无效电平，否则为有效电平。
  • PWM模式2：在向上计数时，TIMx_CNT < TIMx_CCR1时通道1为无效电平，否则为有效电平；在向下计数时，TIMx_CNT>TIMx_CCR1时通道1为有效电平，否则为无效电平。

CAN总线

CAN总线的基本原理

要进行can通讯，需要专门的can收发芯片，单片机通过Tx与Rx与can收发芯片通讯，can收发芯片通过CANH和CANL与can总线通讯。线束上需要双绞线缠绕。

经过can收发器之后，普通信号会转换为差分信号，差分线是用CAN HIGH和CAN LOW两根线来传输信号，低电平下CANH为3.5V，CANL为1.5V，高电平下CANH和CANL均为2.5V。
