Web基础

首先会通过搭建网站，app，云应用的过程来找到这些应用的安全隐患。

Web常规

Web常规包括的四大件：

  • 系统
  • 中间件
  • 数据库
  • 源码

常规的Web应用搭建：

  1. 购买云服务器与域名
  2. 云服务器搭建中间件 （例如windows搭建iis服务器）
  3. 下载并上传Web程序源码
  4. 添加网站并绑定域名目录

子域名模式

通常一个网站会有原域名和主域名，例如：

主域名： 你想了解的“大本营”。比如 example.com。这是你网站最主要、最核心的网址。
子域名： 在大本营（主域名）前面加一个前缀形成的“小分站”。比如：

  • blog.example.com - 博客分站
  • shop.example.com - 商城分站
  • bbs.example.com - 论坛分站
  • admin.example.com - 后台管理分站
  • account.example.com - 用户中心分站

再例如：
www.baidu.com
tieba.baidu.com

tieba就是网站的子域名，baidu.com就是网站的主域名。

子域名模式搭建的网站

通常子域名模式搭建的网站，其安全隐患在于该子域名中所使用的程序漏洞，例如asp程序，php程序等等。

因此从子域名开始进行安全审计和漏洞扫描是非常重要的。

子域名建站的网络安全问题：

  • 攻击面扩大： 每个子域名都是一个独立的“门户”（即使共享主域名）。子域名越多，意味着可能被黑客攻击的“入口”也越多。比如 shop.example.com 商城如果有漏洞，攻击者可能获得用户支付信息或破坏下单；bbs.example.com 论坛如果有漏洞，攻击者可能发垃圾信息或盗取用户账号发言。

  • “短板效应”： 主域名的整体安全性有时取决于 安全性最弱的那个子域名。如果一个保护得很差的 test.example.com (测试站) 被攻破，攻击者可能利用它与主域或其它子域的联系（比如内部DNS记录、内部网络访问）来进一步入侵 admin.example.com 或 shop.example.com。

  • 程序漏洞： PHP 程序 (包括 WordPress, Discuz! 等) 非常流行，但也很容易被黑客盯上，因为它们开源，漏洞容易被发现和利用。如果一个过时的 Discuz! 论坛 bbs.example.com 没有及时修补安全更新，很可能就是黑客入侵整个域名的突破口。

  • ASP/.NET 程序也可能有漏洞，但因为通常更多部署在封闭的企业环境，公开的广泛攻击案例相对PHP程序少一些（不代表更安全！）。而且老旧的 ASP 系统风险尤其高。

  • 配置错误： 子域之间、或者子域和主域之间，如果服务器的配置（SSL证书配置、文件权限、目录访问规则）出错，就可能让黑客从一个点跳到另一个点。

  • 信息收集： 黑客渗透时，会扫遍你所有的子域名和IP，识别上面跑着什么程序和服务（比如 bbs.example.com 用的是 Discuz! X3.4，upload.example.com上有个老旧FTP服务器），目的是找出最脆弱的目标进行攻击。

端口模式

除了子域名外，还可以通过端口模式设置不同的子网站，不同的端口可以通过不同的程序进行搭建：

因此，在后续的网络安全的学习中，端口扫描也是非常重要的，端口越多，攻击点也就越多。

目录模式

在主域名下面的子目录，也可以通过不同的程序构建子网站，同样也是可以攻击的目标。

其他Web构建方式：

  • 集成软件
  • Docker容器
  • 分配站（以博客园为例，用户注册后可得到自己的域名）

源码结构

如果在网站的程序源码中，使用了windows或者类似的身份验证、管理员权限控制等，则作为攻击者无法绕过这些防御。

通过每个web都有属于自己的数据库，用于储存网站的具体内容，并且管理员的账号密码也会在数据库当中。

在源码的数据库配置文件中，会有构建数据库的方式，如Access与MSSQL等，有些数据库需要密码，有些不需要。

当数据库不在web源码的目录下时，我们称之为站库分离。
